# Academic Homepage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a GitHub Pages-compatible bilingual academic homepage with an introduction-first layout, a unified experience timeline, a full publications section, and working `ZH / EN` plus `Light / Dark` switches.

**Architecture:** Implement the site as plain static assets: one `index.html` shell, one stylesheet, one centralized bilingual content module, and one browser module that renders the page and manages UI state. Use CSS variables for light/dark theming, keep all content in a single data source for maintainability, and verify DOM behavior with lightweight automated tests before manual responsive checks.

**Tech Stack:** HTML, CSS, vanilla JavaScript (ES modules), Vitest, jsdom

const STORAGE_KEYS = {
  language: 'homepage-language',
  theme: 'homepage-theme',

function createState() {

    language: localStorage.getItem(STORAGE_KEYS.language) || 'zh',
    theme: localStorage.getItem(STORAGE_KEYS.theme) || 'light',
